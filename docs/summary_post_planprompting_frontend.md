### `docs/specs/safety_policy.md`
New **Rule 11 — Frontend Security** with 6 sub-rules:
- **11.1** API key storage: localStorage only; never in URL (except WSS `?api_key=` param over TLS); clear on 401
- **11.2** Content rendering: all content through `react-markdown` with allowedElements whitelist; `dangerouslySetInnerHTML` forbidden; no `rehypeRaw`
- **11.3** Input validation: chat input is plain text only; frontend never constructs `RunConfig`; client-side param bounds as UX hint only (server remains authoritative)
- **11.4** iframe sandboxing: `allow-scripts allow-same-origin allow-forms` for Streamlit; `allow-scripts allow-forms` for UCSC; no `allow-top-navigation`
- **11.5** WebSocket auth: `?api_key=` validated against `APIKey` table before Redis subscription
- **11.6** No credential exposure in agent responses: strip file paths, AWS credentials, internal IPs, raw stderr before writing to `ChatMessage.content`

### `docs/specs/tool_contracts.md`
New **Section 12: Chat Gateway Interface** defining:
- `ChatDispatchInput`: `conversation_id`, `message_id`, `user_content` (max 4000 chars), `api_key_id`
- `ChatDispatchOutput`: `run_id | None`, `assistant_message_id`, `needs_clarification`, `clarification_prompt | None`
- Full behaviour contract: ambiguous intent → clarification message (no AnalysisRun created); resolvable intent → Pydantic-validated RunConfig → AnalysisRun + arq job
- Redis pub/sub channel naming: `conv:{conversation_id}`
- Constraint: no LLM-generated numerical values written to `AnalysisRun.run_config`
